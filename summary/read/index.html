# 模块化

  • 是什么：模块化是指把一个复杂的系统分解到多个模块以方便编码
  • 优点：解决了以前开发网页需要通过命名空间的方式来组织代码的一些问题

    • 无法合理管理项目依赖和版本
    • 无法方便控制以来的加载顺序

  • 基本使用：
// 导入

# 构建工具

构建工具的作用：（构建其实是工程化，自动化思想在前端开发中的体现）

  • 代码转化：ts 编译成 js，scss 编译成 css
  • 代码优化：压缩 js，css，html代码，压缩合并图片

# Webpack

打包模块化 JavaScript 的工具，一切文件皆模块，通过 Loader 转换文件，通过 Plugin 注入钩子，最后输出由多个模块组成的文件。

  • 基本使用：
module.exports={
    // 入口
    entry:'./app.js',
    // 出口
    output:{
        filename:'bundle.js'
    }
}

  • 核心概念：
    • Entry：入口，Webpack 执行构建的第一步将从 Entry 开始，可抽象成输入。
    • Module：模块，在 Webpack 里一切皆模块，一个模块对应着一个文件。Webpack 会从配置的 Entry 开始递归找出所有依赖的模块。
    • Chunk：代码块，一个 Chunk 由多个模块组合而成，用于代码合并与分割。
    • Loader：模块转换器，用于把模块原内容按照需求转换成新内容。
    • Plugin：扩展插件，在 Webpack 构建流程中的特定时机注入扩展逻辑来改变构建结果或做你想要的事情。
    • Output：输出结果，在 Webpack 经过一系列处理并得出最终想要的代码后输出结果。

# Array.from()

将类数组对象和可遍历(iterable)对象转为真正的数组

// Array.from
let arrayLike = {
    '0': 'red',
    '1': 'green',
    '2': 'blue',
    length: 3
}
// ES5 写法
var arr1 = [].slice.call(arrayLike)
console.log(arr1); // [ 'red', 'green', 'blue' ]

// ES6 写法
console.log(Array.from(arrayLike)); // [ 'red', 'green', 'blue' ]
console.log(Array.from({length:3})); //[ undefined, undefined, undefined ]

# copyWithin()

Array.prototype.copyWithin(target, start = 0, end = this.length)

  • target（必需）：从该位置开始替换数据。如果为负值，表示倒数。
  • start（可选）：从该位置开始读取数据，默认为 0。如果为负值，表示从末尾开始计算。
  • end（可选）：到该位置前停止读取数据，默认等于数组长度。如果为负值，表示从末尾开始计算。
console.log([1,2,3,4,5].copyWithin(1,3)); //[ 1, 4, 5, 4, 5 ]

# find() 和 findIndex()

  • 数组实例的 find方法，用于找出第一个符合条件的数组成员。参数是一个回调函数，所有数组成员依次执行该回调函数，直到找出第一个返回值为 true 的成员，然后返回该成员。如果没有符合条件的成员，则返回 undefined。
    • find方法的回调函数可以接受三个参数，依次为当前的值、当前的位置和原数组
console.log([71,65,59,60,40,38].find((e)=>e>=60)); //71
1
  • 数组实例的findIndex方法的用法与find方法非常类似，返回第一个符合条件的数组成员的位置，如果所有成员都不符合条件，则返回-1。
  • 这两个方法都可以接受第二个参数，用来绑定回调函数的this对象
  • 特点：弥补了 indexOf 无法识别 NaN 的缺点
[NaN].findIndex(y => Object.is(NaN, y)) //0

fill方法使用给定值，填充一个数组。

# entries()，keys() 和 values()

用于遍历数组，它们都返回一个遍历器对象，可以用for...of循环（或使用手动调用遍历器对象的next方法）进行遍历，唯一的区别是keys()是对键名的遍历、values()是对键值的遍历，entries()是对键值对的遍历。

# includes() (ES2016引入)

  • 返回一个布尔值，表示某个数组是否包含给定的值，与字符串的includes方法类似
  • 第二个参数表示搜索的起始位置，默认为0

# flat()，flatMap()

  • flat()用于将嵌套的数组“拉平”，变成一维的数组会跳过空位
  • flat()不管有多少层嵌套，都要转成一维数组
  • flatMap()对原数组的每个成员执行一个函数，然后对返回值组成的数组执行flat()方法，该方法返回一个新数组，不改变原数组
  • flatMap()只能展开一层数组
    • 参数是一个遍历函数，该函数可以接受三个参数，分别是当前数组成员、当前数组成员的位置（从零开始）、原数组。

# 空位处理

ES5 对空位的处理，已经很不一致了，大多数情况下会忽略空位。

  • forEach(), filter(),reduce(), every() 和some()都会跳过空位。
  • map()会跳过空位，但会保留这个值
  • join()和toString()会将空位视为undefined，而undefined和null会被处理成空字符串。

ES6 则是明确将空位转为undefined，由于空位的处理规则非常不统一，所以建议避免出现空位。

# promise相关

ECMAScript 6 入门 Promise 对象

  • 是什么？ Promise是解决异步编程的一种解决方案，诞生于技术社区，ES6将其写入语言标准，统一了用法，原生提供了Promise对象。

  • 为什么？ Promise可以将异步操作以同步操作的流程表达出来，避免了层层嵌套的回调函数，解决了回调地狱问题。

  • 三种状态：

    • 进行中pending
    • 完成fulfilled
    • 失败rejected

  • 特点：

    • 对象状态不受外界影响。如Promise名字一般，作为一个承诺，只接受异步操作结果改变状态。
    • 状态一旦确定，就无法更改，即resolved（已定型），任何时候都可以得到这个结果。

  • 缺点：

    • 必须抛出错误，否则Promise中的错误无法反馈到外部。
    • 一旦新建，他就会立即执行，无法中途取消。
    • 处于pending状态时，无法得知目前进展到哪一个阶段

  • 怎么做？ Promise是一个构造函数，用来生成Promise实例

const promise = new Promise((resolve, reject) => {
    let i=1
    if (i == 1) {
        resolve("成功")
    } else {
        reject("失败")
    }
})

# then方法

可以用来指定不同状态回调函数，两个回调函数作为参数，分别为成功态和失败态使用，then方法返回的是一个新的Promise实例

# catch方法

then(null,rejection)或then(undefined,rejection)的别名，用于指定发生错误时的回调函数

# finally方法(ES2018)

不管状态如何都执行

# all方法

  • 接收一个数组作为参数，将多个Promise实例包装成一个新的Promise实例
  • 全fulfilled，才会变为fulfilled，有一个rejected则为rejected

# race方法

  • 接收一个数组作为参数，将多个Promise实例包装成一个新的Promise实例
  • 率先改变的状态为返回值

# allSettled方法(ES2020)

  • 接收一个数组作为参数，将多个Promise实例包装成一个新的Promise实例
  • 只有当所有状态都返回结果，实例才会结束
const promise1 = Promise.resolve("图片1加载成功");
const promise2 = Promise.reject("图片2加载失败");
const promise3 = Promise.resolve("图片3加载成功");
const promise = Promise.allSettled([promise1, promise2, promise3]);
promise.then((mes) => {
    console.log(mes)
}) 
// [
//   { status: 'fulfilled', value: '图片1加载成功' },
//   { status: 'rejected', reason: '图片2加载失败' },
//   { status: 'fulfilled', value: '图片3加载成功' }
// ]
