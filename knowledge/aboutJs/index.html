# 🍊第五章 变量、作用域和内存问题

# 1.基本类型和引用类型

基本类型存储在栈，引用类型分两部分，栈里存地址，堆里存数据

  • 动态的属性：对引用数据类型可以添加或删除属性和方法
  • 复制变量值：基本数据类型直接传值，引用数据类型传引用的地址
  • 传递参数：函数的传递都是按值传递的，引用数据类型传的是地址，可以把参数想象成局部变量

检测类型

typeof用于检测基本数据类型
检测引用数据类型可以使用原型链来识别（instanceof操作符）

# 2.执行环境及作用域和内存问题

1.简单介绍

  • 执行环境定义了变量或函数有权访问的其他数据，决定了他们各自的行为。
  • 每个执行环境都有一个与之关联的变量对象，环境中定义的所有变量和函数都保存在这个对象中。
  • 每个函数都有自己的执行环境。
  • 当代码在一个执行环境中执行时，会创建变量对象的一个作用域链。用途是保证对执行环境有权访问的所有变量和函数的有序访问。
  • 在作用域链前端，始终都是当前执行代码所在环境的变量对象。
  • 如果这个环境是函数，则将其活动对象作为变量对象
  • 活动对象最初只包含一个变量（即arguments对象）下一个变量对象则来自下一个包含环境，一直延续到全局执行环境（始终为作用域链中最后一个对象）。

TIP

call-apply-bind-用于重定向this

2.延长作用域链 有些语句在作用域链的前端临时增加了一个变量对象，该变量对象的代码会在代码执行后被移除。

  • try-catch语句的catch块
  • with语句

3.块级作用域（类c语言中，由花括号封闭的代码块都有自己的作用域，即执行环境） ES5 只有全局作用域和函数作用域，没有块级作用域（私有作用域）。

  • 内层变量可能会覆盖外层变量。
  • 用来计数的循环变量泄露为全局变量。

ES6的块级作用域

let实际上为 JavaScript 新增了块级作用域。 块级作用域的出现，实际上使得获得广泛应用的匿名立即执行函数表达式（匿名 IIFE）不再必要了。

4.垃圾收集机制（自动）

  • 标记清除（最常用）：进入环境时标记"进入环境"，离开时标记"离开环境"
  • 引用计数（不常用）：跟踪记录每个值被引用的次数，释放次数为0的内存

5.管理内存

  • 全局变量一旦不再使用，设null来释放其引用，叫解除引用。
  • 局部变量离开执行环境时，自动被解除引用
  • 解除一个值的引用并不意味着自动回收该值所占用的内存。解除引用的真正作用是让值脱离执行环境，已便垃圾收集器下次运用时将其回收。

6.内存泄漏

# 2.面向对象

特征：面向对象的编程语言都有类的概念，而通过类可以创建任意多个具有相同属性和方法的对象，ECMAScript中没有类的概念，因此它的对象也与基于类的语言中的对象有所不同

ECMA-262对象的定义:无序属性的集合，其属性可以包含基本值，对象或者函数

# 3.对象的属性类型

ECMA-262第五版在定义只有内部才用的特性时，描述了属性的各种特征，这些特性为了实现JavaScript引擎用的。为了表示特性是内部值，把它们放到了两对儿方括号中，例如[[Enumerable]]

# 1.数据属性

  • [[Configurable]] ：是否能用过delete删除属性，能否修改属性的特性，或者能否把属性改为访问器属性，默认为true。
  • [[Enumberable]] ：能否通过for-in循环返回属性。默认为true
  • [[Writeable]] ：表示能否修改属性的值。默认为true
  • [[Value]] ：包含这个属性的数据值

要修改属性默认的特性需要使用ECMAScript5的 Object.defineProperty() 方法，三个参数：属性所在的对象，属性的名字和描述符对象

# 2.访问器属性

访问器属性不包含数据值，它们包含一对儿setter和getter函数（非必须）,读取访问器属性时会调用getter函数，这个函数负责返回有效的值；在写入访问器属性时，会调用setter函数并传入新值。

访问器属性要通过使用ECMAScript5的 Object.defineProperty() 方法来定义，三个参数：属性所在的对象，属性的名字和描述符对象

  • [[Configurable]] ：是否能用过delete删除属性，能否修改属性的特性，或者能否把属性改为数据属性，默认为true。
  • [[Enumberable]] ：能否通过for-in循环返回属性。默认为true
  • [[Get]] ：在读取属性时调用的函数。默认为undefined
  • [[Set]] ：在写入属性时调用的函数。默认为undefined

TIP

属性名前加下划线(如"_name")是一种常用的记号，表示只能通过对象方法访问的属性。

# 3.定义多个属性（Object.defineProperties）

ECMAScript 5定义了一个 Object.defineProperties() 方法，利用这个方法可以通过描述符一次定义多个属性，接收两个对象参数：第一个对象是要添加和修改其属性的对象，第二个对象的属性与第一个对象中要添加或修改的属性一一对应。

# 4.读取属性的特性（Object.getOwnPropertyDescriptor）

使用ECMAScript 5的 Object.getOwnPropertyDescriptor() 方法，可以取得给定属性的描述符。接收两个参数：属性所在对象和要读取描述符的属性名称。返回值是数据属性或访问器属性组成的对象。

TIP

可以针对任意对象（包括DOM和BOM）
